Robust File Processing: Reading, Counting, and Writing Safely in Python

By Code Daily

Handling files is a common task in programming, but it is fraught with potential errors—missing files, permission issues, or corrupt data. A robust program must not only process data but also anticipate these failures gracefully.

In this tutorial, we write a program that reads a text file, analyzes its contents (counting lines, words, and characters), and writes a summary report to a new file. Crucially, we wrap our logic in a try-except block to ensure the program doesn't crash unexpectedly. 

try:
    # Open the input file in read mode
    file = open("input.txt", "r")

    # Read all data from file
    data = file.read()
    file.close()

    # Processing data
    lines = data.split('\n')
    words = data.split()
    characters = len(data)

    line_count = len(lines)
    word_count = len(words)

    # Open output file in write mode
    out = open("output.txt", "w")

    # Writing processed data to output file
    out.write("Number of lines: " + str(line_count) + "\n")
    out.write("Number of words: " + str(word_count) + "\n")
    out.write("Number of characters: " + str(characters) + "\n")

    out.close()

    print("File processed successfully.")

except FileNotFoundError:
    print("Error: Input file not found.")

except IOError:
    print("Error: File operation failed.")

except Exception as e:
    print("Unexpected error:", e)

By separating specific errors (like FileNotFoundError) from general ones, we can provide specific feedback to the user, making the application much more user-friendly and easier to debug.
