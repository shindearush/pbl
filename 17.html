Mind the Gap: Handling Missing Values with Pandas

By Arush Shinde

Real-world data is rarely perfect. Sensors fail, surveys get skipped, and systems crash, leaving your datasets full of holes. In Pandas, these missing values are represented as NaN (Not a Number). If you attempt to calculate statistics on a dataset filled with NaNs without handling them, your results can be misleading or result in errors.

Data Scientists generally have two main strategies for this: Imputation (filling the gaps) or Deletion (removing the broken records).

In the code below, we simulate missing data using NumPy's np.nan. We then explore fillna(0), which replaces every gap with a zeroâ€”useful for things like sales data where a missing record implies no sale. Alternatively, we show dropna(), the "nuclear option" that discards any row containing even a single missing value. 

import pandas as pd
import numpy as np

# Create DataFrame with missing values (np.nan)
data = {
    "Name": ["Arush", "Riya", "Amit", "Neha"],
    "Marks1": [85, np.nan, 78, 90],
    "Marks2": [80, 88, np.nan, 92]
}

df = pd.DataFrame(data)

print("Original DataFrame:")
print(df)

# ðŸ”¹ Handling missing values using fillna()
df_filled = df.fillna(0)
print("\nDataFrame after fillna():")
print(df_filled)

# ðŸ”¹ Handling missing values using dropna()
df_dropped = df.dropna()
print("\nDataFrame after dropna():")
print(df_dropped)

# ðŸ”¹ Aggregation operations on the filled data
print("\nAggregation Operations:")
print("Sum of Marks1:", df_filled["Marks1"].sum())
print("Mean of Marks1:", df_filled["Marks1"].mean())

print("Sum of Marks2:", df_filled["Marks2"].sum())
print("Mean of Marks2:", df_filled["Marks2"].mean())

Choosing between these methods depends on your domain. If you are calculating the average score of a class, filling a missing test score with 0 penalizes the student (which might be correct if they skipped it). However, dropping the student entirely excludes them from the analysis.
