Enforcing Structure: Implementing Interfaces with Python Abstract Base Classes

By Arush Shinde

In large software systems, it is often necessary to enforce a strict contract between classes. You want to ensure that if a developer creates a new shape, they *must* define how to calculate its area and perimeter. Python achieves this through Abstract Base Classes (ABCs).

In the example below, the Shape class acts as an interface. It defines the methods area and perimeter but does not implement them (indicated by the pass keyword). The @abstractmethod decorator ensures that any class inheriting from Shape (like Rectangle) is forced to provide its own logic for these methods, or else the code will refuse to run. 

from abc import ABC, abstractmethod

# Defining an interface using Abstract Base Class
class Shape(ABC):

    @abstractmethod
    def area(self):
        pass

    @abstractmethod
    def perimeter(self):
        pass

# Implementing the interface
class Rectangle(Shape):

    def __init__(self, length, breadth):
        self.length = length
        self.breadth = breadth

    def area(self):
        return self.length * self.breadth

    def perimeter(self):
        return 2 * (self.length + self.breadth)

# Creating object of implementing class
r = Rectangle(5, 3)

print("Area of Rectangle:", r.area())
print("Perimeter of Rectangle:", r.perimeter())

This pattern is essential for writing clean, maintainable, and error-resistant code, especially when working in teams where multiple developers contribute to the same codebase.
