Beyond Lists: Efficient Data Storage with Python Arrays

By Arush Shinde

While Python lists are incredibly versatile, they aren't always the most memory-efficient choice for storing large amounts of numerical data. This is where the array module comes in. Unlike lists, arrays in Python are constrained to a single data type, making them more compact and faster for certain operations.

In this example, we demonstrate the lifecycle of an array: creation, modification (insertion/deletion), searching, and iteration. Note the use of the type code 'i', which specifies that this array will strictly contain signed integers. 

# Import array module
import array
import time

# Creating an array of integers
arr = array.array('i', [10, 20, 30, 40, 50])
print("Original Array:", arr)

# ðŸ”¹ Insertion
arr.append(60)
print("After Insertion:", arr)

# ðŸ”¹ Deletion
arr.remove(30)
print("After Deletion:", arr)

# ðŸ”¹ Searching
search = 40
if search in arr:
    print(search, "found at index", arr.index(search))
else:
    print(search, "not found")

# ðŸ”¹ Iterating through array
print("Iterating through array:")
for element in arr:
    print(element)

This approach provides a glimpse into lower-level memory management while maintaining Python's readable syntax. It is particularly useful when interfacing with C libraries or handling large datasets where performance is critical.
