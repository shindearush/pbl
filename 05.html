The Blueprint of Code: Understanding Classes and Objects in Python

By Code Daily

Object-Oriented Programming (OOP) allows us to structure programs so that properties and behaviors are bundled into individual objects. Think of a Class as a blueprint (like an architect's drawing), and an Object as the actual house built from that blueprint.

[Image of class vs object diagram]

In the code below, we define a Student class. The __init__ method acts as a constructor—it initializes the object's attributes (Name, Roll No, Marks) the moment an object is created. We then define a custom method, display(), to print these details. 

# Defining a class
class Student:
    
    # Constructor
    def __init__(self, name, roll_no, marks):
        self.name = name
        self.roll_no = roll_no
        self.marks = marks

    # Method to display student details
    def display(self):
        print("Name:", self.name)
        print("Roll No:", self.roll_no)
        print("Marks:", self.marks)

# Creating object of the class
s1 = Student("Arush", 101, 85)

# Calling method
s1.display()

Notice the use of self. In Python, self represents the specific instance of the class. It allows each student object (e.g., s1, s2) to maintain its own unique data without interfering with others.
